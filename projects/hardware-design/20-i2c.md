### Concepts

>[!cite] From [[iic-crash.pdf]]:

>[!cite] [[iic-crash.pdf#page=21]]
>Inter-Integrated Circuit -- 一种同步串行通信接口规范
>IIC总线需要2根连线,时钟线SCL ,数据线SDA,与SPI总线相比,通讯速度  慢很多

>[!cite] [[iic-crash.pdf#page=22]]
>该协议假设发起数据传输的器件为总线**主控器件**, I2C总线上所有的其它器件都被视为总线从器件
>- 主控器件产生时钟,起始信号和停止信号
>- 每个连接到总线的设备都会有一个唯一地址
>- 主设备与从设备之间以字节(8位)为单位进行双向的数据传输
>- 通讯速率: I2C从器件没有最小工作时钟频率,但100KHz通常是基线  

#### I2C Bus timing

```text
Idle     - SDA == 1 &&   SCL == 1
Start    - SDA == F when SCL == 1
(Addr)   - 8-bit (7-bit addr + r/w control)
(Addr ACK) - after receive Addr, slave return low-level ACK. only start transforming after master affirm it. 被选择的 I2C slave 拉低 sda 进行应答
Next bit - SDA changes when SCL == 0
Sampling - SDA stay when SCL == 1
ACK      - After 8-bit (1 byte) release bus when 9th clock
Stop     - SDA == U when SCL == 1
```

**Quick ref**:
- `SCL = 1` means prepared for working
- `SDA` basically run as: `IDLE(1) -> START(F) -> DATAs -> Next(0) -> STOP(U)`

#### Data transform

**MSB**:  for `0b10110010`
```text
MSB -> 1 0 1 1 0 0 1 0 <- LSB
```

### Addressing and R/W Control

In I2C, each peripheral distributed a *7-bit address* with A R/W control bit:
```text
[ 7-bit 从机地址 ][ R/W 位 ]
   7 bits           1 bit
```

- **低 1 位（bit 0）** 是读写控制位：
  - `0` 表示主机**写**数据到从机。  
  - `1` 表示主机从从机**读**数据。  
- **高 7 位（bit 7:1）** 就是从机的 7 位地址。

因此，如果 JY‑901 的 7‑bit 从机地址是 `0x50`（二进制 `1010000`），那么：

- **写地址字节**  
  将 `0x50` 左移 1 位，空出的最低位填 `0`：  
  `0x50 << 1 | 0` = `0xA0` （二进制 `1010000 0`）。

- **读地址字节**  
  将 `0x50` 左移 1 位，最低位填 `1`：  
  `0x50 << 1 | 1` = `0xA1` （二进制 `10100 1`）。
